Developer Preview

AO3 envelope previews and validation states.

This page is for local preview only. Normal AO3 users should start in the user-facing bundle and guides, then come here only if they want to inspect or maintain the local prototype.

Project Constraints

  • AO3 explicitly disallows JavaScript in posted works.
  • AO3 only allows a limited subset of HTML and CSS, but the subset is enough for layered paper effects.
  • The effect is built from stacked blocks, z-index, transform, and transition.
  • Downloads do not keep work skins, and readers can hide custom creator styles.

Touch-friendly

This version exists because hover is awkward on phones and tablets. The visual language stays almost identical, but the state change now comes from a normal link and the CSS :target selector.

AO3 caveat

This is a stronger AO3 experiment because it only uses anchors, IDs, and CSS selectors AO3 work skins already rely on for other interactive fic patterns.

Recommendation

If your audience is mostly desktop readers, publish the hover version first. If mobile readability matters more, use this as the next thing to test in AO3 preview.
