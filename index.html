AO3 Research + Local Prototype

Envelope opening, but within AO3's rules.

This page tries two directions with only HTML and CSS. The first one mirrors the familiar AO3 hover-to-open approach. The second keeps the same visual language but uses links plus :target so the interaction can be tapped instead of hovered.

What the research says

  • AO3 explicitly disallows JavaScript in posted works.
  • AO3 only allows a limited subset of HTML and CSS, but the subset is enough for layered paper effects.
  • The classic envelope effect on AO3 is built from stacked blocks, z-index, transform, and transition.
  • Downloads do not keep work skins, and readers can hide custom creator styles.

Version A is the safe replication path for AO3 because it uses the same hover logic people already use in work skins. Version B is the stronger reading experience if you want touch interaction, and it should be tested in AO3 preview because it relies on link targeting instead of hover.

Touch-friendly

This version exists because hover is awkward on phones and tablets. The visual language stays almost identical, but the state change now comes from a normal link and the CSS :target selector.

AO3 caveat

This is a stronger AO3 experiment because it only uses anchors, IDs, and CSS selectors AO3 work skins already rely on for other interactive fic patterns.

Recommendation

If your audience is mostly desktop readers, publish the hover version first. If mobile readability matters more, use this as the next thing to test in AO3 preview.
